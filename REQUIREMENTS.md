# 需求梳理

本文档根据当前代码实现反向整理，作为后续开发和验收基准。

## 1. 应用定位

每日待办是一个本地优先的桌面待办应用，面向每日任务管理、长期计划跟踪、桌面小组件查看和本地 JSON 数据同步。

## 2. 待办类型

### 2.1 普通待办

- 普通待办绑定到单个日期。
- 普通待办可填写任务文本。
- 普通待办可选时间段，时间段格式为 `HH:mm-HH:mm`。
- 普通待办的完成状态为单一布尔值。
- 普通待办支持删除。
- 普通待办支持到时间段开始时间时提醒。

### 2.2 长期待办

- 长期待办绑定一个开始日期和结束日期。
- 长期待办必须设置有效时间段。
- 长期待办在开始日期到结束日期之间每天显示。
- 长期待办支持按天划分阶段。
- 阶段字段包含开始天数、结束天数和阶段备注。
- 阶段备注不能为空。
- 阶段天数不能重叠，不能超过长期待办总天数。
- 长期待办展示文本包含第几天 / 总天数。
- 如果当天匹配阶段备注，则在展示文本中附带阶段备注。
- 勾选长期待办只完成当前选中日期。
- 长期待办保存 `completedDates` 记录每日完成情况。
- 长期待办支持“提前完成”。
- 提前完成会记录提前完成日期和备注。
- 提前完成后，未来日期不再展示该长期待办。
- 长期待办支持编辑、删除和提前完成，入口在更多菜单中。
- 长期待办支持到时间段开始时间时每日提醒。

## 3. 时间段能力

- 添加待办时使用时间段，而不是单一时间点。
- 时间段必须满足开始时间早于结束时间。
- 最近使用的时间段最多保留 5 条。
- 最近使用时间段来自本地存储和已有待办时间段。
- 历史的上午 / 下午字段仍兼容读取。

## 4. 主窗口

- 主窗口默认显示当前日期待办。
- 顶部可打开/关闭日期选择器。
- 顶部可切换小组件窗口。
- 显示当天完成数量和总数量进度条。
- 添加表单支持普通待办和长期待办切换。
- 当前日期列表只显示该日期可见的待办。
- 普通待办和长期待办使用统一勾选操作，但长期待办按日期记录。

## 5. 日期选择器

- 日期选择器按月展示。
- 不允许选择未来日期。
- 不允许翻到当前月份之后。
- 日期格显示普通待办完成圆点。
- 如果当天存在长期任务，则不显示普通圆点，改为长期任务综合状态。
- 长期任务综合状态规则：
  - 蓝色：该日期长期任务均已完成。
  - 红色：该日期存在未完成长期任务。
  - 白色：该日期为今天或未来，尚未结算。
- 日期格显示长期任务数量，例如 `长3`。
- 点击日期后，日历保持打开，并更新下方详情区。
- 详情区展示选中日期的总待办完成情况。
- 详情区展示当天长期任务列表，包括任务名、第几天/总天数、阶段备注、当天状态和整体进度条。

## 6. 小组件窗口

- 小组件是独立 Tauri 窗口。
- 小组件无边框、透明、置顶、可调整大小、跳过任务栏。
- 小组件展示今日待办、今日日期、完成进度。
- 小组件每 5 秒重新加载待办数据。
- 小组件每分钟刷新当前日期。
- 小组件可点击待办完成或取消完成。
- 小组件中长期待办显示分段进度背景。
- 小组件中文本单行省略，完整内容通过系统 tooltip 查看。
- 小组件大面积可拖动，待办项点击不会触发窗口拖动。

## 7. 长期待办进度颜色

- 长期待办进度按自然日分段。
- 当前日期之前的日期才结算完成/未完成。
- 已完成日期显示蓝色。
- 未完成日期显示红色。
- 今天及未来显示为未结算状态。
- 主列表、小组件和日历详情使用一致的颜色语义。

## 8. 数据存储和同步

- 应用默认使用本地应用数据目录保存 `todos.json` 和 `settings.json`。
- 用户可指定一个数据文件夹。
- 指定文件夹后，应用固定使用该文件夹下的 `daily-todos.json`。
- 如果 `daily-todos.json` 存在，切换后自动加载该文件。
- 如果 `daily-todos.json` 不存在，切换后使用当前待办创建默认文件。
- 保存数据时，如果目标文件已存在，会先生成同目录 `.bak` 备份。
- 保存数据时使用临时文件写入，再复制到目标文件。
- 用户可重新加载当前数据文件，适合 Git pull 后手动同步。
- 用户可恢复默认本地数据位置。
- 应用不直接执行 Git 操作，外部笔记仓库同步由用户自行管理。
- 保存格式为 JSON 数组，元素为待办对象。

## 9. 通知

- 普通待办在当天时间段开始时通知一次。
- 长期待办在可见日期的时间段开始时通知一次。
- 长期待办通知按 `notifiedDates` 避免当天重复提醒。
- 通知使用 Tauri notification 插件。

## 10. 系统托盘和窗口行为

- 应用创建系统托盘。
- 托盘菜单包含显示编辑窗口、切换小组件和退出。
- 左键点击托盘图标显示编辑窗口。
- 关闭编辑窗口时隐藏窗口，不退出应用。
- 托盘退出才结束应用。

## 11. 开机启动

- 底部提供开机启动开关。
- Windows 使用当前用户 Run 注册表项。
- Linux 使用 XDG autostart desktop entry。
- macOS 使用用户级 LaunchAgent plist。

## 12. 兼容性

- 旧字段 `timeSlot`、`customTime`、`notified` 保留读取兼容。
- 旧设置字段 `dataFilePath` 保留读取兼容，新配置使用 `dataFolderPath`。

## 13. 桌面平台

- 应用基于 Tauri 2，理论支持 Windows、Linux 和 macOS。
- Windows 可在本机直接构建 exe、NSIS 和 MSI。
- Linux 包需要在 Linux 环境构建。
- macOS 包需要在 macOS 环境构建。
- 推荐使用 GitHub Actions 在各平台 runner 上生成对应产物。
